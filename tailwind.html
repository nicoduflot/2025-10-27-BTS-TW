Installation et lancement

Installer Tailwind

Mode Client

Il est possible d'installer tailwind sur différents frameworks.

Dans ce projet, nous allons l'installer sur un projet sans framework, en utilisant Tailwint CLI

  1. Il faut utiliser un terminal et se rendre à la racine du projet.
  2. Une fois à cet endroit, il faut utiliser npm pour laner la commande suivante : npm install tailwindcss @tailwindcss/cli
  3. Dans le projet, prévoir un fichier css d'entrée, par exemple à cet endroit : ./src/tailwind/intput.css, sur lequel on inscrit la ligne suivante : 
                    
@import "tailwindcss";

Lancement

npx

  1. A nouveau dans le terminal, on va lui demander de "surveiller" le fichier input, et d'écrire le css compilé par tailwind sur un autre fichier css, ici il sera dans ./src/css/style.css, en utilisant la commande suivante : npx @tailwindcss/cli -i ./src/tailwind/input.css -o ./src/css/style.css --watch
  2. Une fois lancé, le terminal va recompiler le css à chaque fois que les fichiers dans lesquels sont lié le fichier style.css seront sauvegardés. Il ajoutera le CSS final dans ce fichier, selon les classes écrite dans le fichier HTML.
                    
<!doctype html>>
    <html lang="fr">>
        <head>>
            <meta charset="UTF-8">
            <meta name="viewport" content="width=device-width, initial-scale=1.0">
            <link href="./src/css/style.css" rel="stylesheet">
        </head>
    <body>
        <h1 class="text-3xl font-bold underline">Hello world!</h1>
    </body>
</html>

Attacher le fichier de sortie

Il suffit ensuite d'attacher le fichier de sortie à la page qui doit appliquer le CSS généré par Tailwind, quand on "surveille" le fichier d'entrée, Tailwind surveille aussi le fichier qui appelle le fichier de sortie. Tout changement dans le fichier HTML ou le fichier de d'entrée va relancer la compilation du fichier de sortie.

Modèle colonne Vs Modèle Grille

Utiliser les colonnes

La classe columns-<nombre>

En utilisant la classe columns-x (où x est le nombre de colonnes désiré), on va définir le nombre d'éléments dans l'odre des affichage d'écran, en commençant par l'affichage smartphone et en remontant jusqu'aux écrans plus larges.

<section class="mb-1 columns-1 lg:columns-2 xl:columns-3 gap-1">

La classe break-<options>

Demander des colonnes va, par défaut, "casser" les éléments pour qu'ils aient tous la même hauteur. Pour éviter cela, il faudra ajouter par exemple aux éléments à l'intérieur la classe break-inside-avoid-column

<article class="mb-1 rounded-sm p-2 border border-gray-900 break-inside-avoid-column">

Gérer le gap horizontal

On applique la classe gap-x-<nombre> à l'élément qui va contenir les éléments de la grille, selon la valeur ajoutée, on jouera sur l'espace horizontal entre les éléments à l'intérieur.

<section class="mb-1 columns-1 lg:columns-2 xl:columns-3 gap-1">

Gérer le gap vertical

Avec le système columns, la classe gap ne contrôle que l'espacement horizontal entre les colonnes (l'équivalent de column-gap en CSS).

Il faut pour cela régler ajouter la classe gap-1 au conteneur et une classe de marge extérieure basse mb-1 ou my-1 par exemple pour établir un gap vertical.

<article class="mb-1rounded-sm p-2 border border-gray-900 break-inside-avoid-column">

Pas de span

Le système de colonnes dans Tailwind CSS (columns-*) est basé sur la propriété CSS columns, qui est conçue pour créer des mises en page de type magazine/journal avec du contenu qui s'écoule naturellement d'une colonne à l'autre. Ce n'est pas un système de grille.

Le problème

Le système columns ne permet pas de faire "span" un élément sur plusieurs colonnes de la même manière que Grid. C'est une limitation du CSS multi-column layout lui-même, pas de Tailwind.

Utiliser les grilles

La classe générale grid

On applique la classe générale grid à l'élément qui va contenir les éléments de la grille, ensuite il faut configurer le nombre de colonnes à l'intérieur, en commençant par l'affichage smartphone et en remontant jusqu'aux écrans plus larges.

<div class="py-2 grid">

La classe grid-cols-x

On applique la classe grid-cols à l'élément qui va contenir les éléments de la grille, pour configurer le nombre de colonnes souhaitées dans l'élément selon les différents points de ruptures.

<div class="py-2 grid grid_cols-1 lg:grid-cols-2 xl:grid-cols-3">

La classe gap-x

On applique la classe gap-<nombre> à l'élément qui va contenir les éléments de la grille, selon la valeur ajoutée, on jouera sur l'espace entre et au-dessus/en-dessous des éléments à l'intérieur.

<div class="py-2 grid grid_cols-1 lg:grid-cols-2 xl:grid-cols-3 gap-1">

La classe col-span

Pour qu'un élément prenne x colonnes de large, il faut ajouter à cet élément une classe col-span-x, où x est le nombre de colonnes.

L'article précédent récupère les classes :
<div class="p-2 lg:col-span-2 xl:col-span-1 rounded-sm border border-gray-900">

<div class="py-2 grid grid_cols-1 lg:grid-cols-2 xl:grid-cols-3 gap-1">

Il faut préciser la constrution des colonnes pour que les éléments soient cohérent, surtout qu'une fois qu'une classe applique un nombre de colonne, tant qu'on ne lui indique pas le contraire, elle garde cette configuration.

L'article actuel récupère donc les classes suivantes :
<div class="p-2 lg:col-span-2 xl:col-span-3 rounded-sm border border-gray-900">
